## What?
- It is a message broker that implements the Advanced Message Queuing Protocol 
- It achieves this by using Producers, Consumers and Brokers. 
- Then helps with loose coupling as well as scalability. 

## Players:
- Producer: publishes the message to the consumer
- Consumer: consumes the message sent by the producer
- Exchange: Middleman between the producer and the queue. The producer does not directly interact with the queue. Compares the 'routing key' and the 'binding key' 
- Queue: Holds the messages the producer has for the consumer

## Communication paths:
- The exchange and queue connect with one another by a 'binding' and they use a binding key for this purpose. 
- The producer along with a message sends a routing key to the exchange. How an exchange compares the two keys depends on the type of exchange it is 

## Types of Exchange:
- Fanout: routing key ignored, the message is sent to all the queues
- Direct: routing key = binding key, these queues receive the message 
- Topic: Allows a partial pattern match 
- Header: Uses the header message instead of the routing key
- Default (nameless): compares the routing key with the queue name and forwards the message accordingly. This special exchange is directly implemented by RabbitMQ
