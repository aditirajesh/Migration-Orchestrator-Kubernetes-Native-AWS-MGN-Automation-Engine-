#### This table contains a list of all possible states that a server in this orchestration tool can take as well as reasons for failure

| S.No | State                                                                 | Reasons for Failure                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ---- | --------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1    | PENDING                                                               | **The ADD_SERVER handler calls mgn.describe_source_servers(). It can fail because:    **                                                                               <br>  - Server not found in MGN at all — agent was never installed on the source machine<br>  - Server found but in DISCONNECTED lifecycle state — agent was installed but lost connectivity to the MGN endpoint (firewall rule changed, agent crashed, network outage)<br>  - Wrong aws_source_server_id passed in the job payload — human data entry error at registration time<br>  - MGN API authentication failure — credentials expired or missing mgn:DescribeSourceServers permission<br>  - MGN regional endpoint unreachable — AWS service disruption |
| 2    | AGENT_INSTALLED                                                       | **mgn.update_replication_configuration() can fail because:**<br>  - Staging subnet ID doesn't exist or is in a different VPC than expected<br>  - Replication security group ID doesn't exist or was deleted<br>  - The specified replication instance type (t3.small etc.) is not available in the target AZ<br>  - No route from the staging subnet to the MGN endpoint (VPC routing misconfiguration)<br>  - EC2 quota exceeded — can't launch the replication server<br>  - Missing IAM permissions for ec2:CreateNetworkInterface or similar replication setup calls                                                                                                                                                              |
| 3    | REPLICATION_STARTED                                                   | **mgn.start_replication() and the subsequent poll can fail because:**<br>  - MGN failed to launch the replication server — capacity issue in the staging AZ<br>  - Initial sync never starts — source server blocked egress to MGN endpoint on port 1500<br>  - Replication server launched but staging disk fills up — undersized staging disk type<br>  - Source server agent disconnects mid-sync — OS reboot, patch window, network instability<br>  - Replication lag never reaches zero — source disk write rate exceeds replication bandwidth (busy OLTP server)<br>  - Poll exceeds MAX_POLL_ATTEMPTS — replication taking longer than expected                                                                                |
| 4    | READY_FOR_TESTING / TEST_LAUNCH_TEMPLATE_CONFIGURED                   | **mgn.update_launch_configuration() can fail because:**<br>  - Target subnet doesn't exist or has no available IPs<br>  - Security groups specified don't exist in the target VPC<br>  - IAM instance profile referenced in the launch template doesn't exist<br>  - Instance type specified is not available in the target AZ<br>  - Invalid or missing post-launch script configuration                                                                                                                                                                                                                                                                                                                                              |
| 5    | TEST_INSTANCE_LAUNCHING / TEST_INSTANCE_RUNNING                       | **mgn.start_test() and job polling can fail because:**<br>  - MGN job fails immediately — launch template configuration is invalid at execution time (even if the API accepted it during configuration)<br>  - Insufficient EC2 capacity for the specified instance type<br>  - Target subnet has no available IPs<br>  - Boot volume encryption key not accessible by the launch role<br>  - Instance launches but OS never boots (could be a grub issue) corrupt conversion or incompatible kernel<br>  - MGN job exceeds timeout<br>  - Tag error (if tag is mentioned for spot instances this could throw an error)                                                                                                                |
| 6    | AWAITING_TEST_VALIDATION                                              | This is a human gate — the only automated failure here would be infrastructure-level (RabbitMQ goes down while waiting, database connection lost). If the engineer rejects, that goes through the approval service directly to FAILED — not via the rollback worker.                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| 7    | TEST_FINALIZED                                                        | **mgn.terminate_target_instances() can fail because:**<br>  - Test instance was already terminated manually by someone — MGN may error or silently succeed depending on state<br>  - Missing mgn:TerminateTargetInstances permission<br>  - MGN API transient error                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 8    | CUTOVER_LAUNCH_TEMPLATE_CONFIGURED / AWAITING_CUTOVER_LAUNCH_APPROVAL | Same failure modes as the test launch template stage. Human gate carries the same caveat as AWAITING_TEST_VALIDATION.<br>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 9    | CUTOVER_STARTED / CUTOVER_SYNC_IN_PROGRESS                            | **mgn.start_cutover() and job polling can fail because:**<br>  - Final delta sync fails — source server disconnected during the sync window<br>  - Final sync takes longer than MAX_POLL_ATTEMPTS allows — high write rate on source<br>  - Cutover instance launch fails — same capacity/configuration reasons as test instance<br>  - Source server agent disconnects precisely during the final sync — worst case scenario, data in unknown state                                                                                                                                                                                                                                                                                   |
| 10   | CUTOVER_INSTANCE_LAUNCHING → AWAITING_CUTOVER_VALIDATION              | Same failure modes as test instance, but consequences are more severe. Instance in unknown partially-booted state. Always results in FROZEN.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 11   | CUTOVER_FINALIZED onwards                                             | Any failure here is FROZEN by definition. The production instance is already running and the source is being disconnected — there is no automated undo. Human must assess the actual state of both the source and destination before taking any action.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
