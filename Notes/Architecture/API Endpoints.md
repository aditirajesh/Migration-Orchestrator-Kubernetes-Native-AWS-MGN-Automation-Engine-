### Batch Pipeline Actions

All batch pipeline action endpoints apply the same operation to all eligible servers in the batch simultaneously. A server is **skipped** if it is not in the required state — this is not an error, just a gate not yet ready for that server. A server **fails** only if the state advance or job dispatch raises an exception. Every response contains three lists: `succeeded`, `skipped`, `failed`.
